# 背景  
在OSX系统使用中文输入法的过程中，会莫名切换到ABC输入法，这时需要手动切换回中文输入法，而ABC输入法(英文)又无法直接删除，这产品设计挺恶心的。  
  
# 解决  
网上有很多教程删除ABC输入法，参考[教程](https://www.jianshu.com/p/0ba1292441b9)，通过修改系统SIP属性(CMD+R进入恢复模式，csrutil disable关闭SIP)，然后下载一个[PlisteditPro](https://fatcatsoftware.com/plisteditpro/PlistEditPro_187.zip)，从一个plist文件中删除ABC输入法。  
该方法确实可以删除ABC输入法，只留下中文输入法，但不幸的是，我遇到了一个bug：电脑重启后会提示输入用户密码，登陆后电脑短暂卡住，进入锁屏界面，再次提示输入密码，但输入密码时无法输入小写u，不管怎么捣鼓都没法输入，但密码中又含有u，此时已经进入了死锁，电脑无法打开了。  
在网上搜索了一圈，发现有人遇到过类似的[情形](https://www.macx.cn/thread-2075486-1-1.html)，谢天谢地，按大神方法成功打开电脑，不用重装系统、重装软件，感谢大神分享。  
重启系统，启动时一直按住CMD+R进入OSX恢复模式，在菜单栏-实用工具中找到终端，然后输入命令resetpassword重置密码，不含小写u。重启系统，进入正常登陆界面，会提示钥匙串信息，选择“创建新的钥匙串”即可。之后在输入法管理中将英文输入法添加回来，系统恢复正常。  
  
# 总结  
不要删除默认的英文输入法，系统不让删除肯定是有(xue)道(de)理(jiao)的(xun)，无厘头直接就让你无法登陆，最后还是给加回来才正常了。  
另外可以设置系统切换输入法的快捷键，如control+space，这样也许可以缓解下你的困扰。
